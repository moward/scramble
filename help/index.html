About the Game

This game is a Java application written by Matthew Howard for a CIS 120 homework assignment at the University of Pennsylvania. This original game concept is a combination of a platform game and a word game. The player either collects letters to spell English words (Freestyle) or to spell out an answer to some question (Quiz Mode).

How to Play

Goal

The goal of the game is to win as many points as possible by correctly spelling and submitting words before time runs out.

Game Objects

Game object include:

The player: The player is controlled by the user. He can walk left, walk right, and jump. Additionally, he can control his motion while in mid-air. He can also collect the below game objects.

Letters: Letters move through the screen from left to right throughout the game and can be collected by the player to spell words. The distribution of letters depends on the play mode.

Erasers: Erasers will erase the most recently spelled letter. Multiple erasers may be used in a row.

Gameplay

Game play begins when the user selects a play mode. Then he or she has 240 seconds (4 minutes) to accumulate as many points as possible by spelling words. The player spells words by collecting letters and then submitting the built words by hitting "Enter". There is no way for the player to die. However, the user may start a new game at any time either by hitting Ctrl+N or selecting "New Game" from the "Game" menu.

Play Modes

Freestyle

In freestyle, the player can try to spell out any words in the English dictionary. The game's dictionary has approximately has approximately 58,000 words. Words can be any length longer than 1.

Scoring: Each letter has its own point value inversely proportional to the square root of its frequency in the English language. More exactly, the point value is Math.ceil(Math.sqrt(1/f/13)) where f is the decimal frequency. This data comes from Letter frequency - Wikipedia.

The total point value for a word is the sum of the letter points times a length multiplier. This multiplier is worth the length of the word divided by 3 rounded up. A word not in the dictionary or a skip results in a 20 point penalty.

Examples:
"an" is 1*(1+2)
"computer" is 3*(2+2+2+2+2+1+1+2) = 42

Letter distribution: Letters are distributed randomly according to their frequency in the English language. In other words, each letter is the result of a weighted random. This data also comes from Letter frequency - Wikipedia.

Quiz

In quiz mode, the player is given questions or prompts and must spell out the answer correctly to get points. All answers must be spelled omitting spaces. The three included question packs are State Capitals, Chemical Elements, and Super Bowl winners. For State Capitals, users are prompted with a state and must spell the capital of that state without the spaces. For chemical elements, the user is given a chemical symbol (like "Ag") and must spell the element name ("silver"). For Super Bowl winners, the user is given a Super Bowl number (like "XLVII") and must spell the city (or region for the Patriots) of the winning team ("Baltimore").

Scoring: A correct answer is worth 10 points for each letter in the word. An incorrect answer or a skip results in a 10 point penalty.

Letter distribution: Half of all letters are completely random, with each letter having an equal probability of appearing. The other half are letters from the answer. These letters are weighted using a piecewise function so that every word has at least a one-part chance of showing up. The length of the spelled letters is used to determine what the next correctly spelled letter would be. The next letter in the correctly spelled word has the greatest chance of showing up, with each letter surrounding it having one fewer part chance of showing up.

For example, if the correct answer were "Orange" and the played spells "App", each new letter would have a half chance of being any letter in the alphabet and additionally a 1/15 chance of being a "o", 2/15 of a "r", a 3/15 of a "a", a 4/15 of a "n", a 3/15 of a "g", and a 2/16 of a "e".

Game Features

 The following consideration and features went into making this game:
  • Real physics: The code in this game uses accurate physics for acceleration due to gravity, vertical acceleration, and friction between the player and platform.
  • Graphics: All of the graphics and sprites for this game were custom-made. The player is animated by four sprites, all drawn in Photoshop. The eraser and letter icons were made from scratch. Furthermore, all of the colors used on the field are based off of a popular color scheme (Elianne3).
  • Sound effects: This game includes 5 different custom-made 8-bit sound effect made on Bfxr.
  • The Dictionary class from Homework 9: Yup! The Freestyle mode uses the dictionary class from Homework 9, only with a small change in how it loads the dictionary file.
  • Swing time compensation: The timer provided by the Swing library is fairly inaccurate. If it runs slows or fast, the events in the game take place at the same rate, with just the framerate changing. This way, the system's speed does not effect gameplay beyond rendering quality.
